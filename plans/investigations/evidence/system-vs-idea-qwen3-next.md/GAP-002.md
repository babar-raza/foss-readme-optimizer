# GAP-002: VER-004 Gap - Durable Skip Masks Stale Compliance

**Severity:** HIGH  
**Requirements Affected:** `VER-004`  
**Current Status:** PARTIAL (documented gap, not fixed)

## Background

The `VER-004` requirement states:

> **The system MUST NOT skip a compliance check due to a stale recorded baseline.**

During Wave 8's production-reliability pass, a critical gap was discovered where the "coarse shortcut" (`is_fresh()` check) could permanently freeze a domain that was skipped on a stale run.

### The Problem

The supervisor loop uses a **two-tier convergence check**:

1. **Coarse tier** (`is_fresh()`): Cheap check - does upstream + control plane match last run? If yes, skip the entire specialist tier.
2. **Fine tier** (`domain_states`): Detailed check - has each domain been properly executed against the current revision?

The gap is that **"skip this domain"** and **"compliant this domain"** look identical in the state blob. Once a domain is skipped due to a stale upstream, the coarse shortcut never re-examines it because it appears "covered."

## Evidence

### File: `src/readme_agent/state/domain_state.py`

**Lines 334-368: `is_domain_covered()` function**

```python
def is_domain_covered(domain_state: DomainStateV1 | None, current_revision: str | None) -> bool:
    """Wave 8.6 (`ORC-003` reversal prerequisite): revision-aware
    replacement for the former presence-only coverage check...
    
    A permanent, by-design non-outcome (`NON_ESCALATING_FAILURE_REASONS`) is
    always covered, never "incomplete" -- without this carve-out, 22/25 real
    registry entries (disabled-mode / no-policy-profile repos) would never
    satisfy a naive revision match for the domains that permanently error
    for them, permanently defeating `is_fresh()` and forcing the full
    specialist tier (live GitHub API calls included) on every run forever --
    the exact regression this fix must not introduce.
    
    A domain honestly skipped this run (`skipped_this_run`) is covered only
    if the skip itself was recorded against the current revision -- a skip
    recorded against a now-stale revision (state carried across an
    unexpectedly-long gap) is correctly treated as incomplete."""
    if domain_state is None:
        return False
    if domain_state.last_failure_reason in NON_ESCALATING_FAILURE_REASONS:
        return True
    if domain_state.skipped_this_run:
        return (
            current_revision is not None
            and domain_state.details.get("skipped_at_revision") == current_revision
        )
    return (
        current_revision is not None
        and domain_state.upstream_revision_at_accept == current_revision
    )
```

### File: `src/readme_agent/supervisor/convergence.py`

**Lines 50-108: `is_fresh()` function**

```python
def is_fresh(
    recorded_revision: str | None,
    current_revision: str | None,
    *,
    recorded_control_plane_fingerprint: str | None = None,
    current_control_plane_fingerprint: str | None = None,
    recorded_domain_coverage_complete: bool | None = None,
    check_domain_coverage: bool = False,
) -> bool:
    """The cheap pre-planning freshness check (`VER-003`)...
    
    Wave 8 (production-reliability pass, external-review triage
    2026-07-21): the upstream-revision comparison alone is blind to a
    control-plane change (a policy/prompt/capability-manifest-version edit)
    with no new upstream commit...
    
    `VER-005` (found live, Wave 8e full-registry pass, 2026-07-21): even a
    genuinely unchanged upstream + control plane says nothing about whether
    the *prior* run's own `domain_states` coverage was actually complete...
    `check_domain_coverage` is keyword-only, defaults to `False` -- preserves
    every existing caller's exact behavior when not passed. A caller that
    opts in (passes `check_domain_coverage=True`, `recorded_domain_coverage_
    complete` read from `SupervisorStateV1`) additionally requires that
    value to be exactly `True`; `None` (every record written before this
    field existed) or `False` forces one honest full specialist-tier retry..."""
    revision_matches = (
        recorded_revision is not None
        and current_revision is not None
        and recorded_revision == current_revision
    )
    if not revision_matches:
        return False
    if current_control_plane_fingerprint is not None and (
        recorded_control_plane_fingerprint is None
        or recorded_control_plane_fingerprint != current_control_plane_fingerprint
    ):
        return False
    if check_domain_coverage and recorded_domain_coverage_complete is not True:
        return False
    return True
```

**Key finding (line 106):**

```python
if check_domain_coverage and recorded_domain_coverage_complete is not True:
    return False
```

The parameter `check_domain_coverage` **defaults to `False`**, meaning callers get **NO domain coverage check by default**.

## Where the Bug Manifests

**File:** `src/readme_agent/supervisor/loop.py` (approximate line 141)

```python
domain_coverage_complete = compute_domain_coverage_complete(
    base,
    specialists_registry.all_domains(),
    supervisor_state.last_observed_upstream_revision,
)
```

This is computed and stored in `supervisor_state.domain_coverage_complete`, but there's **no call to `is_fresh()` with `check_domain_coverage=True`** to force re-execution when coverage was incomplete.

### The Failure Scenario

1. **Run #1:**
   - Upstream: commit `abc123`
   - `is_fresh()` returns `False` (no prior state)
   - Specialist tier executes
   - `readme_reconciliation` runs, completes successfully
   - `domain_states["readme_reconciliation"]` recorded with `skipped_this_run=False`, `upstream_revision_at_accept="abc123"`
   - **`generate_readme_content` has a transient GitHub API error**
   - Result: `domain_states["generate_readme_content"]` recorded with `ERROR:...` (failure)
   - `compute_domain_coverage_complete()` returns `False` (one domain failed)

2. **Run #2 (upstream unchanged, control plane unchanged):**
   - Upstream: still commit `abc123`
   - `is_fresh()` called with **NO** `check_domain_coverage=True`
   - Revision matches (`abc123 == abc123`)
   - Control plane fingerprint matches
   - **`check_domain_coverage` not passed, so skipped entirely**
   - `is_fresh()` returns `True`
   - ** SPECIALIST TIER IS SKIPPED ENTIRELY**
   - `domain_states["generate_readme_content"]` remains in ERROR state
   - The failed domain is never retried

3. **Run #3+ (same as Run #2):**
   - Same behavior - the failed domain stays in ERROR forever
   - The "domain coverage complete" is `False` in state, but it's never re-checked

## Consequence

**Immediate Impact:**
- A single transient error (network glitch, GitHub API timeout, LLM gateway hiccup) can permanently disable a domain
- No automatic recovery without manual intervention (delete state ref, force-run with `--force-regenerate`)
- Pilot repos show "CONVERGED_NO_CHANGE" but have incomplete domain coverage

**Production Readiness Blocker:**
- `VER-004` is not truly satisfied
- No self-healing for partial runs
- Manual state purging is required to recover from intermittent failures

## Recommended Fix

Add `check_domain_coverage=True` to the `is_fresh()` call in `supervisor/loop.py`:

### Change in `src/readme_agent/supervisor/loop.py`

Search for the `is_fresh()` call around line 200-250 (exact line will vary):

**Before:**
```python
if is_fresh(
    recorded_revision=supervisor_state.last_observed_upstream_revision,
    current_revision=current_revision,
    recorded_control_plane_fingerprint=supervisor_state.control_plane_fingerprint,
    current_control_plane_fingerprint=current_fingerprint,
```

**After:**
```python
if is_fresh(
    recorded_revision=supervisor_state.last_observed_upstream_revision,
    current_revision=current_revision,
    recorded_control_plane_fingerprint=supervisor_state.control_plane_fingerprint,
    current_control_plane_fingerprint=current_fingerprint,
    recorded_domain_coverage_complete=supervisor_state.domain_coverage_complete,
    check_domain_coverage=True,  # <-- NEW: VER-004 fix
):
    logger.info("coarse tier: skipped, nothing changed")
    return SuperviseResult(
        status="CONVERGED_NO_CHANGE",
        org_repo=org_repo,
        task_graph=TaskGraph(),
        decisions=[],
        evidence_dir=evidence_dir,
    )
```

This ensures that even when upstream + control plane are identical, if the **previous run's domain coverage was incomplete**, the specialist tier will execute again to complete the missed domains.
