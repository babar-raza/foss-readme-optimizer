# GAP-001: No Continuous Scheduling

**Severity:** HIGH  
**Requirements Affected:** `OPS-004` (Phase 18)  
**Current Status:** PLANNED

## Background

The `OPS-004` requirement states:

> **The golden-set monitor MUST run on a schedule and manual trigger, use the LLM-exercising pilot, remain hard-coded to dry-run, and append stable run history.**

This enables **continuous monitoring** of the system's output quality without manual intervention. Without scheduled execution, the system requires a human to manually trigger workflows on GitHub Actions, which defeats the purpose of an "autonomous" agent.

## Evidence

**Analysis:** This workflow does have scheduled execution, but it runs `supervise` across the **entire allow-list** (all 25+ repos), not a golden-set pilot. The `OPS-004` requirement specifically calls for "golden-set monitor" - a small, carefully-selected subset of repos that exercise the LLM frequently for quality monitoring.

## Consequence

**Immediate Impact:**
- No automatic quality monitoring without human intervention
- Each run requires someone to click "Run workflow" in GitHub UI
- Cannot establish baseline drift over time without manual scheduling
- Workflow run history is incomplete (only triggered runs are recorded)

**Production Readiness Blocker:**
- Phase 18 cannot be completed (OPS-004 explicitly unmet)
- No objective evidence of continuous operation
- Golden-set monitoring claim is not operational

## Recommended Fix

Add a **dedicated golden-set monitoring workflow** that:

1. **Runs on schedule** (e.g., `cron: "0 * * * *"` for hourly, or `cron: "0 8,12,18 * * *"` for daily business hours)
2. **Targets only golden-set repos** (3 pilots with LLM exposure, not the full 25-repo portfolio)
3. **Remains hard-coded dry-run** (no `mode: "full"` option)
4. **Uploads evidence artifacts** for historical analysis
5. **Uses the supervisor entry point** (not the legacy `run` command)

### Example Implementation

Create a new file: `.github/workflows/readme-agent-golden-set-monitor.yml`

```yaml
name: README Agent Golden-Set Monitor (dry-run)

# Phase 18: OPS-004 -continuous monitoring of golden-set pilot repos
# Runs hourly during business hours to provide regular quality snapshots
# without overwhelming the system or the LLM gateway.
on:
  schedule:
    - cron: "0 8,12,18 * * 1-5"  # Hourly 8am-6pm Mon-Fri
  workflow_dispatch:

permissions:
  contents: read

jobs:
  monitor:
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false  # Don't cancel other runs if one fails
      matrix:
        repo_key:
          - aspose-cells-foss/Aspose.Cells-FOSS-for-Java
          - aspose-3d-foss/Aspose.3D-FOSS-for-Java
          - aspose-pdf-foss/Aspose.PDF-FOSS-for-Java
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: pip install -e .
      - run: readme-agent supervise --repo "${{ matrix.repo_key }}" --durable-state
        env:
          GH_TOKEN: ${{ secrets.GH_TOKEN }}
          LLM_BASE_URL: ${{ secrets.LLM_BASE_URL }}
          LLM_API_KEY: ${{ secrets.LLM_API_KEY }}
      - id: safe_repo_key
        run: echo "value=$(echo '${{ matrix.repo_key }}' | sed 's#/#__#')" >> "$GITHUB_OUTPUT"
      - uses: actions/upload-artifact@v4
        with:
          name: "golden-monitor-${{ steps.safe_repo_key.outputs.value }}-${{ github.run_id }}"
          path: runs/evidence/
          retention-days: 90
```
